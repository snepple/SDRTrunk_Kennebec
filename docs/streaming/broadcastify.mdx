SDRTrunk Kennebec supports two distinct Broadcastify integrations that serve different listener experiences. **Broadcastify Feed** streams continuous live audio to the Broadcastify network in real time so listeners can tune in at any moment, while **Broadcastify Call** uploads each completed trunked call recording as a discrete MP3 file with metadata to the Broadcastify Calls API. You can enable one or both depending on how your Broadcastify account is provisioned.

## Broadcastify Feed vs. Broadcastify Call

| | Broadcastify Feed | Broadcastify Call |
|---|---|---|
| **Delivery model** | Continuous live stream | Per-call upload after call ends |
| **Protocol** | Icecast TCP (compatible with Icecast 2.3.2) | HTTPS POST to Broadcastify Calls API |
| **Audio format** | MP3, 16 kbps, mono, 8000 Hz (automatic) | MP3 |
| **Listener experience** | Tune in at any time, hear audio now | Browse individual call recordings with metadata |
| **Required credentials** | Host, port, mount point, password, feed ID | API key, system ID |

## Setting Up Broadcastify Feed

Broadcastify Feed delivers a live radio stream to the Broadcastify network. Listeners hear decoded audio in near real time through the Broadcastify website or mobile app.

### Prerequisites

Log in to your Broadcastify account and locate the feed you want to connect. Gather the following values from the feed's configuration page:

- Hostname
- Port
- Mount point (for example, `/stream` or a feed-specific path)
- Source password
- Feed ID (a numeric identifier assigned to your feed)

### Add the Broadcaster

<Steps>
  <Step title="Open the Streaming Editor">
    In SDRTrunk Kennebec, go to **View** > **Streaming**.
  </Step>
  <Step title="Add a Broadcastify Feed broadcaster">
    Click **+** and select **Broadcastify Feed**.
  </Step>
  <Step title="Enter the connection details">
    Fill in the following fields in the configuration panel:

    | Field | Description |
    |---|---|
    | **Name** | A label for this configuration, for example `County Fire Main` |
    | **Host** | The Broadcastify server hostname |
    | **Port** | The server port number |
    | **Mount Point** | The stream path, starting with `/` |
    | **Password** | The source password for your feed |
    | **Feed ID** | Your numeric Broadcastify feed ID |
  </Step>
  <Step title="Enable verbose logging (optional)">
    Toggle **Verbose Logging** on if you need detailed diagnostic output in the application log. This is useful for troubleshooting connection or audio quality issues.
  </Step>
  <Step title="Enable and save">
    Check **Enabled**, then click **Save**. SDRTrunk Kennebec connects to the Broadcastify server immediately and begins streaming decoded audio.
  </Step>
</Steps>

<Note>
  SDRTrunk Kennebec sets the audio parameters for Broadcastify Feed automatically: 16 kbps MP3, mono, 8000 Hz. You do not need to configure bit rate or sample rate.
</Note>

## Setting Up Broadcastify Call

Broadcastify Call is a completed-call push service. When each trunked call ends, SDRTrunk Kennebec encodes the recording as an MP3 file and uploads it to `https://api.broadcastify.com/call-upload`. The call then appears in the Broadcastify Calls interface with talkgroup, timestamp, and other metadata.

### Prerequisites

Log in to your Broadcastify account and locate the Calls API key and system ID for your system. These are separate from your feed credentials.

### Add the Broadcaster

<Steps>
  <Step title="Open the Streaming Editor">
    In SDRTrunk Kennebec, go to **View** > **Streaming**.
  </Step>
  <Step title="Add a Broadcastify Call broadcaster">
    Click **+** and select **Broadcastify Call**.
  </Step>
  <Step title="Enter the configuration">
    Fill in the following fields:

    | Field | Description |
    |---|---|
    | **Name** | A label for this configuration |
    | **API Key** | Your Broadcastify Calls API key |
    | **System ID** | Your numeric Broadcastify system ID |
    | **Host** | Leave at the default production endpoint (`https://api.broadcastify.com/call-upload`) unless directed otherwise |
  </Step>
  <Step title="Enable and save">
    Check **Enabled**, then click **Save**. SDRTrunk Kennebec uploads completed call recordings to the Broadcastify Calls API as they arrive.
  </Step>
</Steps>

<Tip>
  A development endpoint (`https://api.broadcastify.com/call-upload-dev`) is available for testing. Switch the **Host** field to this value while verifying your configuration, then switch back to the production endpoint before going live.
</Tip>

## Verifying the Connection

<Tabs>
  <Tab title="Broadcastify Feed">
    After enabling the broadcaster, open **View** > **Broadcaster Status**. The Feed entry should show **Connected** within a few seconds.

    You can also visit your feed's Broadcastify page while audio is being received to confirm the stream is active.
  </Tab>
  <Tab title="Broadcastify Call">
    After enabling the broadcaster and receiving a trunked call, the **Broadcaster Status** panel shows **Connected** once the first upload attempt succeeds. Check your Broadcastify Calls interface to confirm new call entries are appearing.
  </Tab>
</Tabs>

## Troubleshooting

If the **Broadcaster Status** panel shows **Disconnected** or **Error**, work through the following checklist:

- Verify the hostname, port, and mount point exactly match the values shown in your Broadcastify feed settings.
- Check that the password has no leading or trailing spaces.
- Confirm the outbound port is open on your network (commonly port `80` or `8000`).
- Check that your Broadcastify account is active and the feed is provisioned.

Enable **Verbose Logging** on the Broadcastify Feed configuration and review the application log for detailed error messages.

<Warning>
  The **Delay** and **Maximum Recording Age** fields are available in the configuration but should be left at their defaults unless Broadcastify support specifically advises you to change them. Setting **Maximum Recording Age** too low can cause call recordings to be discarded before they are uploaded.
</Warning>
