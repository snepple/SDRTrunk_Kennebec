📏 Ergo: Fix NBFMConfigurationEditor Unintended Save Prompts

💡 What:
Added missing `mLoadingConfiguration` checks to multiple UI listeners in `NBFMConfigurationEditor` and properly toggled the flag when overriding `setItem()`. Also added a safety check to `ChannelConfigurationEditor`'s base `EditorModificationListener`.

🎯 Why:
To prevent the editor's `modified` property from being incorrectly set to `true` when a channel's configuration is merely being loaded into the UI fields (e.g., right after saving, or when selecting a different channel), which previously caused annoying "Save Changes" prompts.

📸 Before/After:
Before: Selecting a channel, editing, saving, and clicking another channel triggered a "Save Changes" prompt even though it was just saved.
After: Switching channels after saving works seamlessly without extraneous prompts.

🧠 Cognitive Load:
Reduces user frustration and confusion by eliminating false-positive save prompts, ensuring the application trusts that the user has already saved their work.
