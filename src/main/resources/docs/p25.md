# Decode P25 Phase 1 and Phase 2 trunked systems

APCO Project 25 (P25) is the dominant digital radio standard for public safety in North America. SDRTrunk Kennebec supports both Phase 1 and Phase 2, letting you monitor fire, law enforcement, and EMS trunked systems with a single SDR receiver. This page explains what differentiates the two phases and walks you through setting up a P25 channel in the **Playlist Editor**.

## Phase 1 vs. Phase 2

### Phase 1

P25 Phase 1 uses C4FM (Compatible 4-Level Frequency Modulation) or CQPSK (LSM) modulation at **9,600 bps**. Each radio frequency carries a single voice or data channel with a **12.5 kHz** channel bandwidth. Phase 1 is the most widely deployed P25 variant and is found on the vast majority of public safety systems built before the mid-2010s.

SDRTrunk automatically decodes both C4FM and the linear-simulcast (LSM/CQPSK) variant. You do not need to select the modulation type manually — the decoder probes both.

### Phase 2

P25 Phase 2 uses HDQPSK modulation with Time Division Multiple Access (TDMA), fitting **two simultaneous voice channels** onto a single **12.5 kHz** carrier at an effective bit rate of **12,000 bps** per timeslot. Phase 2 systems are common on larger metro and statewide networks.

Because Phase 2 uses TDMA, SDRTrunk spawns two independent decoder timeslots (Timeslot 1 and Timeslot 2) per traffic channel frequency. The decoder can auto-detect the scramble (randomizer) parameters, or you can provide them manually if auto-detection fails.

> [!NOTE]
> P25 Phase 2 traffic channels are granted by a Phase 1 control channel. You always configure the **control channel as Phase 1** in the **Playlist Editor**, even on a Phase 2 system. SDRTrunk detects the Phase 2 upgrade automatically from the control channel signaling.

## Audio decoding and JMBE

P25 uses the IMBE (Phase 1) and AMBE+2 (Phase 1 and Phase 2) voice codecs. To decode audio from these codecs, SDRTrunk Kennebec requires the **JMBE** library, which is **not bundled** with the application. JMBE must be installed separately due to patent considerations around the AMBE codec.

> [!WARNING]
> Without the JMBE library installed, SDRTrunk Kennebec will decode all P25 signaling and display talkgroup activity in the event log, but **no audio will play**. Visit the SDRTrunk wiki for instructions on building and installing JMBE.

## Set up a P25 channel

**1. Open the Playlist Editor**

Select **View → Playlist Editor** from the main menu. Choose the playlist you want to edit, or create a new one.

**2. Add a new channel**

Click **New** in the Channels toolbar and select **P25 Phase 1** from the dropdown. A new channel entry appears in the table and the detail editor opens below.

**3. Set the protocol**

The **Decoder** field is pre-set to **P25 Phase 1**. If your system's control channel is natively Phase 2 on its own carrier (rare), select **P25 Phase 2** instead.

**4. Enter the control channel frequency**

Type the control channel frequency in the **Frequency** field in Hz. For example, enter `851125000` for 851.125 MHz.

**5. Configure the traffic channel pool**

The **Traffic Channel Pool Size** controls how many simultaneous voice calls SDRTrunk will decode. The default is sufficient for most systems. Increase it on busy metro systems; lower it on resource-constrained hardware.

Enable **Ignore Data Calls** to skip decoding data-only grants and conserve CPU resources.

**6. Enable NAC filtering (optional)**

The Network Access Code (NAC) is a 12-bit value (0–4095) that identifies a P25 system. Enable **NAC Filter** and add the allowed NAC values to reject messages from adjacent systems that appear on the same frequency.

**7. Set a Talkgroup Override (optional)**

When monitoring a conventional P25 channel, you can assign a specific **Talkgroup To Assign** (1-65535) directly in the channel configuration. This overrides the dynamically decoded talkgroup from the signal, allowing you to force a fixed alias lookup or streaming rule for all audio produced by this channel. Leave this field blank to use the decoded talkgroup.

**8. Assign system and site names**

Enter a **System** name and optional **Site** name so the channel appears correctly in the **Now Playing** view and in recording filenames.

**9. Save and start decoding**

Click **Save**. Select the channel in the table and click **Play**, or enable **Auto-Start** so it begins every time SDRTrunk launches.


## Talkgroup management

P25 talkgroups are 16-bit numeric identifiers (0–65535) extracted from every voice grant. Without an alias, they appear as raw numbers in the **Now Playing** panel. Assign aliases to give them meaningful names.

**10. Open the Aliases tab**

In the **Playlist Editor**, click **Aliases** in the sidebar.

**11. Create an alias group**

Click **Add Group** and name it to reflect the agency or function, such as `Dispatch` or `Fire`.

**12. Add talkgroup aliases**

Inside the group, click **Add Alias**. Set the **Protocol** to `APCO-25` or `APCO-25 P2`, enter the talkgroup number in the **Identifier** field, and type a display name.

**13. Add radio ID aliases (optional)**

P25 supports individual radio IDs. Add aliases with the identifier type set to **Radio ID** to tag transmissions from specific subscriber units.


> [!TIP]
> Many public safety systems are listed on RadioReference.com with pre-built talkgroup lists. Use **Radio Reference** in the Playlist Editor sidebar to import talkgroups directly rather than entering them by hand.

## Audio equalization

The P25 decoder includes a 5-band graphic equalizer you can enable per channel to compensate for voice codec artifacts on heavily loaded sites. In the channel's **Decoder** tab, enable **Graphic EQ** and adjust the five bands at 100 Hz, 400 Hz, 1 kHz, 2 kHz, and 3.5 kHz to taste.

## Common issues

**Control channel locked but no voice calls play**

Confirm that **Traffic Channel Pool Size** is greater than zero. Also verify that your tuner's capture bandwidth is wide enough to cover the traffic channel frequencies. Calls granted to frequencies outside the tuner's range will not decode.

**Audio sounds choppy or robotic**

This usually indicates an SDR sample rate or USB bandwidth issue. Try lowering the tuner gain, reducing the number of active channels, or switching to a USB 3.0 port. On Phase 1 sites using linear simulcast (LSM), enable the **LSM** modulation mode if C4FM decoding fails.

**Phase 2 talkgroups show garbled IDs**

Phase 2 uses scramble parameters tied to the system's WACN and System ID. Enable **Auto-Detect Scramble Parameters** in the Phase 2 decoder configuration. If auto-detection does not work within a few minutes, look up the WACN and System ID on RadioReference and enter them manually in the **Scramble Parameters** field.

**Messages from neighboring systems appear in the event log**

Enable **NAC Filter** and add only the NAC(s) for your target system. SDRTrunk will discard messages that do not match the configured NAC values.

**Control channel active but unaliased talkgroups do not play**

If you enabled **Ignore Unaliased Talkgroups**, only talkgroups with an alias entry are decoded. Either disable that option or add alias entries for the talkgroups you want to hear.
