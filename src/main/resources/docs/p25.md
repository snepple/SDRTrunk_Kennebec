# Decode APCO-25 P25 trunked radio systems in Kennebec

> Configure SDRTrunk Kennebec to decode APCO-25 Phase 1 and Phase 2 trunked radio systems for public safety monitoring and talkgroup filtering.

APCO Project 25 (P25) is the dominant digital radio standard for public safety in North America. SDRTrunk Kennebec supports both Phase 1 and Phase 2, letting you monitor fire, law enforcement, and EMS trunked systems on the same hardware with no additional hardware or software. This guide walks you through what differentiates the two phases and how to get a P25 channel running in the Playlist Editor.

## Phase 1 vs. Phase 2

  ### Phase 1
    P25 Phase 1 uses C4FM (Compatible 4-Level Frequency Modulation) or CQPSK (LSM) modulation at **9,600 bps**. Each radio frequency carries a single voice or data channel. Phase 1 is the most widely deployed P25 variant and is found on the vast majority of public safety systems built before the mid-2010s.

    SDRTrunk decodes both C4FM and the linear-simulcast (LSM/CQPSK) variant automatically. You do not need to select the modulation type manually — the decoder probes both.

    **Channel bandwidth:** 12.5 kHz


  ### Phase 2
    P25 Phase 2 uses HDQPSK modulation with Time Division Multiple Access (TDMA), fitting **two simultaneous voice channels** onto a single 12.5 kHz carrier at an effective bit rate of **12,000 bps** per timeslot. Phase 2 systems are increasingly common on larger metro and statewide networks.

    Because Phase 2 uses TDMA, SDRTrunk spawns two independent decoder timeslots (Timeslot 1 and Timeslot 2) per traffic channel frequency. The decoder can optionally auto-detect the scramble (randomizer) parameters, or you can provide them manually if auto-detection fails.

    **Channel bandwidth:** 12.5 kHz


> **Note:**
  P25 Phase 2 traffic channels are granted by a Phase 1 control channel. You always configure the **control channel as Phase 1** in the Playlist Editor, even on a Phase 2 system. SDRTrunk detects the Phase 2 upgrade automatically from the control channel signaling.

## Set up a P25 channel

  ### Open the Playlist Editor
    From the main menu, select **View → Playlist Editor**. Choose the playlist you want to edit, or create a new one.


  ### Add a new channel
    Click **Add Channel** (the + icon in the Channels panel). A new channel entry appears in the table.


  ### Set the protocol
    In the channel configuration panel, open the **Decoder** drop-down and select **P25 Phase 1**. If your system's control channel is Phase 2 on its own carrier (rare), select **P25 Phase 2** instead.


  ### Enter the control channel frequency
    Type the control channel frequency in the **Frequency** field in Hz (for example, `851,125,000` for 851.125 MHz).


  ### Configure the traffic channel pool
    The **Traffic Channel Pool Size** setting controls how many simultaneous voice calls SDRTrunk will decode. The default is sufficient for most systems. Increase it on busy metro systems; decrease it on low-memory machines.

    Enable **Ignore Data Calls** to skip decoding data-only grants and save resources.


  ### Enable NAC filtering (optional)
    The Network Access Code (NAC) is a 12-bit value (0–4095) that identifies a P25 system. Enable **NAC Filter** and add the allowed NAC values to reject messages from adjacent systems that appear on the same frequency.


  ### Assign a system and site name
    Enter a **System** name and optional **Site** name so that the channel appears correctly in the Now Playing view and in recording file names.


  ### Save and start decoding
    Click **Save**. Select the channel in the table and click the **Play** button, or enable **Auto-Start** so it starts every time SDRTrunk launches.


## Talkgroup management

P25 talkgroups are 16-bit numeric identifiers (0–65535) that SDRTrunk extracts from every voice grant. Without an alias, talkgroups appear as raw numbers. Assign aliases to give them meaningful names.

  ### Open the Alias Editor
    In the Playlist Editor, select the **Aliases** tab.


  ### Create an alias group
    Click **Add Group** and name it (for example, `Dispatch` or `Fire`). Alias groups help organize large systems.


  ### Add talkgroup aliases
    Inside the group, click **Add Alias**. Set the **Protocol** to `APCO-25` or `APCO-25 P2`, enter the talkgroup number in the **Identifier** field, and type a display name.


  ### Add radio ID aliases (optional)
    P25 supports individual radio IDs. Add aliases with the **Protocol** set to `APCO-25` and the identifier type set to **Radio ID** to tag transmissions from specific radios.


> **Tip:**
  Many public safety systems are listed on RadioReference.com with pre-built talkgroup lists. Import a RadioReference export file directly into the Playlist Editor to populate aliases without manual entry.

## Audio quality options

The P25 decoder includes a 5-band graphic equalizer you can enable per channel to compensate for voice codec artifacts on heavily loaded sites. In the channel's **Decoder** tab, enable **Graphic EQ** and adjust the five bands (100 Hz, 400 Hz, 1 kHz, 2 kHz, and 3.5 kHz) to taste.

## Common issues

  ### The decoder locks onto the control channel but no voice calls play
    Check that your **Traffic Channel Pool Size** is greater than zero. Also confirm that the tuner's bandwidth is wide enough to cover both the control channel and the traffic channels. If traffic channels appear on frequencies outside the tuner's capture range, they will not decode.


  ### Audio is choppy or sounds like robot voice
    This usually indicates an SDR sample rate or USB bandwidth issue. Try lowering the tuner gain, reducing the number of active channels, or switching to a USB 3.0 port. On P25 Phase 1 sites using linear simulcast (LSM), enable the **LSM** modulation mode if C4FM decoding fails.


  ### Phase 2 talkgroups show garbled IDs
    Phase 2 uses scramble parameters tied to the system's WACN and System ID. Enable **Auto-Detect Scramble Parameters** in the Phase 2 decoder configuration. If auto-detection does not work within a few minutes, look up the WACN and System ID on RadioReference and enter them manually in the **Scramble Parameters** field.


  ### Messages from neighboring systems appear in the event log
    Enable **NAC Filter** and add only the NAC(s) for your target system. SDRTrunk will then discard messages that do not match the configured NAC values.


  ### The control channel shows activity but unaliased talkgroups do not play
    If you enabled **Ignore Unaliased Talkgroups**, only talkgroups with an alias entry will be decoded. Either disable that option or add alias entries for the talkgroups you want to hear.
